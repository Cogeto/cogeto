## Fixes that landed in Part A

### F8 — deterministic relative dates (decision 0007 ruling 1)

`temporal-resolver.ts` (chrono-node + a custom F8 pass) resolves raw expressions
against the note anchor in UTC: next-weekday-strictly-after, "by X" → valid_until,
"in N days/weeks" → add, unresolvable → null + recorded in
`memory.temporal_unresolved` (migration 0007, surfaced in the drawer). The
extractor contract gains `temporal_expressions`; the resolver accepts both the
new form and v0001's pre-resolved ISO fields, so nothing breaks before the v0002
prompt (S3.5-B). 10 unit tests encode the three F8 triples exactly plus edge
cases (anchor-on-weekday, month/year boundaries).

### F6 — one citation grammar, strip the rest (decision 0007 ruling 2)

Canonical form `{{cite:<uuid>}}` in `@cogeto/shared/citations.ts`. The backend
post-processor maps the model's `[F#]` markers (incl. comma clusters) to
canonical cites and strips every other bracket/brace token, counting
`citation_violation` (metadata only, logged). The SPA renderer trusts only the
canonical form. Documented in `project/prompts/answer/README.md`.
`citation_never_leaks` proves `[F2, F4]`, malformed braces, and unknown-id cites
never render.

### Model tiers (decision 0007 ruling 3)

Gateway maps `pipeline` (extraction/verification, `mistral-small-latest`) and
`answer` (chat synthesis + eval grader, `mistral-medium-latest`); embeddings
unchanged. Task sites request a tier, never a model. `.env.example` documents
both with a cost note (only the low-volume answer path uses the pricier model).

### Incidental: gateway retry/backoff

`MistralModelGateway.call()` now retries retryable failures (429/5xx/network)
with exponential backoff before throwing — required for the call-heavy evals and
a genuine hardening of the fast path against transient rate limits. Strengthens
§A.10; no Addendum decision affected.

and under-retrieve). Entity, hedge, no-mechanics, and citation checks pass now

citation checks pass.

Observation for Part B: retrieval fuses vector neighbours, so an unrelated
question rarely returns zero facts — the answer model declines on relevance
rather than hitting the canned zero-retrieval path. The decline is correct; the
canned path is effectively reserved for a truly empty store.
