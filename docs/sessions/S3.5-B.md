## What shipped

### §1 extraction/v0002 (F7, F8 contract) + admission rule

- New output contract: `temporal_expressions` (raw phrases + kind — the model no
  longer computes dates; code resolves them, decision 0007 ruling 1), a per-fact
  `hedged` boolean + `hedge_phrase`, and a `subject_entity` (the ONE entity the
  fact is ABOUT, distinct from mentions).
- Zod schema extended (defaults keep older output valid); pipeline + registry
  switched to `extraction/v0002` and `verification/v0002`.
- **Admission rule:** `active` only when `hedged=false` AND verifier `supported`;
  a hedged fact is `uncertain` even when supported. `subject_entity` stored on
  `memory` (migration 0008); `hedge_phrase` stored on `verification_result`.

### §2 verification/v0002 (F7 calibration)

Judges **support only** — never downgrades for tentative wording (hedging is the
extractor's dimension). Two embedded contrast examples: a plainly-stated summary
is `supported` (the Petra pattern, guarding F7 false-positives) and a faithfully
carried hedge is `supported`. Adds "a correctly resolved relative date is not an
addition". v0001 independence rules kept.

### §3 conversational query rewriting (F3)

`query_rewrite/v0001` + `retrieval/query-rewrite.ts`: one bounded pipeline-tier
call resolves pronouns/ellipsis in the latest turn against the last 6 turns into
a self-contained query + entities. Skipped for self-contained, non-trivial turns
(cheap guard); times out to the raw query. Named test **`pronoun_rewrite`**
asserts the rewriter fires for "who is she?" and its output drives retrieval.

### §4 entity-profile retrieval mode (F1, F4, F5)

`retrieval/entity-profile.ts` + refactored `RetrievalService.retrieve` (now
returns `{ memories, mode, focusEntity }`):

- **Entity-profile** (deterministic: a who-is/tell-me-about question with exactly
  one focus entity) → **exhaustive gather** of that entity's memories (all
  entity-search matches for its name variants + vector hits that actually concern
  it — never an unrelated neighbour, the F1 trap), deduped, ordered by status
  weight then recency, capped at 50, no top-k truncation.
- **Project/topic aggregation** (F5): when default results cluster on a dominant
  entity, widen once via entity search before answering.

### §5 answer/v0002 (F1, F2, F4, F6, hedge display)

- **Describe the world, not the retrieval** — the words facts/records/referenced
  are forbidden in user-visible text (F2).
- **subject + profile mode** — a `MODE` block and per-fact subject; in
  `entity_profile` mode the answer opens with who the subject is and enumerates
  ALL their facts; a fact ABOUT Ana that MENTIONS Marta describes Ana, and a
  mentioned person is never presented as the asked-about person (F1/F4).
- **Citations** — the strict `[F#]`→`{{cite:uuid}}` grammar with few-shot
  examples and an explicit ban on `[F2, F4]` clustering (F6).
- **Hedge display** — uncertain facts included with soft framing, never as
  confirmed. Honest-gap behavior unchanged. Chat runs on the answer tier.

## Before / after — golden set (`npm run eval`, 27 cases)

| run | precision | recall | verification agreement |
|---|---|---|---|
| S3.5-A baseline (v0001) | 80.5% | 88.2% | 72.0% |
| S3.5-B (v0002), run A | 85.4% | 94.1% | 84.0% |
| S3.5-B (v0002), run B | 80.5% | 88.2% | 80.0% |

Every aggregate metric is at or above the baseline on both runs — no regression.
Per language, the win is concentrated where the failures were: **English
verification agreement rose from ~57–67% to 88.9–94.4%** (v0002 calibration:
Petra supported, faithfully-carried hedges supported, resolved dates not
additions). English extraction stayed ~88–92% precision / ~91–96% recall.

**Honest caveat (Croatian).** hr verification agreement is stable at 57.1% under
v0002 (both runs), down from the S3.5-A single sample of 85.7% (which itself sat
in a noisy 50–86% band under v0001). hr *extraction* improved (precision
66.7→73.3%, recall 81.8→90.9%). The likely cause is that v0002's calibration
contrast examples are English-only; a `verification/v0003` with Croatian contrast
examples is queued. This dip ships with this explanation (per the golden-set
doc's philosophy) because the aggregate improved and the session's target
(English/the Ana persona) is strongly and stably up.

## Cost note (answer-tier pricing)

Per decision 0007 ruling 3, the **answer tier** (`mistral-medium-latest`) now
serves chat synthesis and the eval grader; the high-volume **pipeline tier**
(`mistral-small-latest`) still serves extraction/verification, and embeddings are
unchanged. So the pricier model is confined to the low-volume, user-facing
interactive path — one answer generation per chat turn, plus one *small-tier*
rewriter call on contextual turns (skipped for self-contained turns). Ingestion
and embedding costs (the bulk of token volume) are unmoved. Net: quality is
raised where the user reads output, without moving the ingestion/embedding cost
floor.
